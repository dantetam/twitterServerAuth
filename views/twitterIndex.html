Welcome to Twitter Server, a RESTful API for retrieving, querying, and analyzing large groups of tweets using natural language processing.

This is a full-stack MongoDB, express, node.js server for experimentation with async node.js, RESTful APIs, and MongoDB storage and querying.


Retrieving and Storing Tweets

Look up tweets and Twitter users. /twitter/tweetAndUserLookup

Start up the Twitter server. /twitter   With topic parameter. /twitter/:topic
Search a topic and dynamically deliver a word map visualization. /twitter/wordmap/:topic
Search live tweets and group by topic associations. /twitter/topicgroups

Querying and Analyzing Tweets

Search the tweet data. /twitterData   With query parameter. /twitterData/:query
Search most recent tweets. /twitterData/recent
Search most recent tweets and use cluster approximation algorithm. /twitterData/recentcluster
Search most recent tweets and use MST algorithm. /twitterData/recentmst
Better topic grouping algorithm (overlap). /twitterData/topicgroups
Query a large amount of tweets and visualize their words. /twitterData/wordmap
Analyze tweets for words after a chosen topic. /twitterData/wordlookup/:searchTweetWord/:inspectWord
Analyze recent tweets for sentiment. /twitterData/recentsentiment

Twitter Users

Add a user's Twitter timeline (tweets) to the database. /twitter/user/:screenName
Query tweets by user. /twitterData/user/:screenName
Find a user's 'sentiment vector.' /twitterData/userSentiment/:screenName
Compare users' sentiment vectors. /twitterData/userSentiment/:screenName

Deeper Tweet Analysis / Advanced Search

Query a large corpus (document set) of tweets. /twitterData/corpus
Query a large set of topics from the above corpus. /twitterData/corpusTopics
Find commonly associated proper nouns. /twitterData/topicAssociations
Advanced search with query parameter. /twitterData/advSearch/:topic
Advanced clustering tweet search. /twitterData/clustersTweetSearch

Other RESTful Features / Authentication

Get current database status. /count  
Test the login screen and database. /login
